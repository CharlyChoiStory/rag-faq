## ✅ Phase 1 — 기본 동작 (다음주 목표)

> 목표: 일단 챗봇이 답변을 내뱉는 것!

### 구성 요소
| 항목 | 선택 | 이유 |
|------|------|------|
| 임베딩 | `jhgan/ko-sroberta-multitask` | 한국어 특화, 무료 |
| 청킹 | FAQ Q&A 단위 분리 | FAQ 구조에 최적 |
| 검색 | 기본 벡터 검색 (코사인 유사도) | 구현 단순 |
| LLM | Claude API | 한국어 자연스러움 |
| UI | Streamlit 기본 챗봇 | 빠른 구현 |

### 완료 기준
```
✅ Notion 데이터 추출 성공
✅ 벡터 DB에 FAQ 저장 성공
✅ 질문 입력 → 답변 출력 동작
✅ Streamlit 화면에서 대화 가능
```

---

## 🔄 Phase 2 — 검색 품질 향상

> 목표: 더 정확하게 찾게 만들기!

### 추가 요소
| 항목 | 선택 | 효과 |
|------|------|------|
| Hybrid Search | BM25 + 벡터 검색 융합 | 검색 정확도 ↑↑ |
| Multi-Query | 질문을 3가지 버전으로 확장 | 누락 문서 감소 |

### 완료 기준
```
✅ "환불" 질문에 "반품" 관련 문서도 검색됨
✅ 짧은 질문도 잘 찾음
✅ Phase 1 대비 답변 정확도 체감 향상
```

---

## 🏆 Phase 3 — 고품질 완성

> 목표: 실제 서비스 수준으로!

### 추가 요소
| 항목 | 선택 | 효과 |
|------|------|------|
| Re-ranking | Cohere Rerank 또는 BGE | 정밀도 최고 수준 |
| Parent-Child 청킹 | 검색=작은단위, 답변=큰단위 | 문맥 풍부 |
| Contextual Compression | 관련 부분만 추출 | 토큰 절약 |

### 완료 기준
```
✅ 모호한 질문도 정확히 답변
✅ 답변 출처 명확히 표시
✅ 응답 속도 3초 이내
✅ 유튜브 데모 시연 가능 수준 🎬
```
