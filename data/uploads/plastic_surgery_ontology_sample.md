# 소규모 성형외과용 실무형 온토로지 샘플

이 문서는 **소규모 성형외과에서 FAQ/RAG 챗봇에 활용할 수 있는 실무형 온토로지 샘플**입니다.  
목적은 복잡한 학술적 온토로지가 아니라, 챗봇이 고객 질문을 보고 다음을 판단하게 만드는 것입니다.

> “이 질문은 예약 문의인가?”  
> “비용 문의인가?”  
> “수술 후 일반 문의인가?”  
> “부작용 의심 질문인가?”  
> “즉시 병원 연결이 필요한가?”

---

## 1. 소규모 성형외과용 온토로지 기본 구조

성형외과 챗봇용 온토로지는 크게 5개로 구성하면 됩니다.

| 구분 | 역할 |
|---|---|
| 시술 분류 | 눈성형, 코성형, 필러, 보톡스 등 구분 |
| 질문 유형 | 예약, 비용, 준비사항, 수술 후 관리, 환불 등 |
| 증상 분류 | 붓기, 멍, 통증, 출혈, 시야 이상 등 |
| 위험도 | 낮음, 중간, 높음, 긴급 |
| 대응 규칙 | 일반 답변, 상담실 연결, 의료진 확인, 응급 안내 |

핵심은 다음 5개를 연결하는 것입니다.

> **시술명 + 질문유형 + 증상 + 위험도 + 대응방법**

## 3. 챗봇이 실제로 어떻게 판단하는지 예시

### 예시 1. 일반 예약 문의

사용자 질문:

> “오늘 쌍수 상담 가능해요?”

온토로지 판단:

```json
{
  "procedure": "눈성형",
  "question_type": "예약문의",
  "risk_level": "low",
  "action": "예약 가능 시간 확인 또는 상담실 연결"
}
```

답변:

> 오늘 눈성형 상담은 예약 상황에 따라 가능합니다. 희망 방문 시간과 연락처를 남겨주시면 상담실에서 가능 시간을 확인해 드릴 수 있습니다.

---

### 예시 2. 비용 문의

사용자 질문:

> “코수술 얼마예요?”

온토로지 판단:

```json
{
  "procedure": "코성형",
  "question_type": "비용문의",
  "risk_level": "medium",
  "action": "예상 범위 안내 후 상담 필요 안내",
  "forbidden": "확정 금액 단정 금지"
}
```

답변:

> 코성형 비용은 콧대, 코끝, 보형물 사용 여부, 재수술 여부, 마취 방식에 따라 달라질 수 있습니다. 대략적인 범위는 안내 가능하지만, 정확한 비용은 상담 후 수술 방법이 결정된 뒤 안내됩니다.

---

### 예시 3. 수술 후 일반 문의

사용자 질문:

> “쌍수하고 멍이 있는데 괜찮나요?”

온토로지 판단:

```json
{
  "procedure": "눈성형",
  "question_type": "수술후관리",
  "symptom": "멍",
  "risk_level": "medium",
  "action": "일반 안내 + 위험 신호 확인"
}
```

답변:

> 눈성형 후 멍은 생길 수 있고 시간이 지나며 완화되는 경우가 많습니다. 다만 심한 통증, 갑작스러운 부기, 출혈, 시야 이상이 함께 있으면 즉시 병원으로 연락해 주세요.

---

### 예시 4. 위험 질문

사용자 질문:

> “필러 맞고 입술이 하얗게 변했어요.”

온토로지 판단:

```json
{
  "procedure": "필러",
  "question_type": "부작용의심",
  "symptom": "피부색 변화",
  "risk_level": "high",
  "action": "즉시 병원 연락 안내",
  "forbidden": "정상이라고 단정 금지"
}
```

답변:

> 필러 후 입술이나 피부색이 하얗게 변하거나 심한 통증이 있으면 빠른 확인이 필요합니다. 지금 바로 병원으로 연락해 주세요. 연락이 어렵거나 증상이 심하면 가까운 응급실을 이용해 주세요.

```text
사용자 질문 입력
↓
온토로지로 시술명·질문유형·위험도 1차 분류
↓
벡터 DB에서 관련 FAQ 검색
↓
검색된 FAQ + 온토로지 규칙을 함께 LLM에 전달
↓
답변 생성
↓
위험도가 high이면 일반 답변 대신 즉시 병원 연결 안내
```

추천 순서는 다음과 같습니다.

> **온토로지로 먼저 위험도를 보고, 벡터 DB로 관련 FAQ를 찾고, 마지막에 안전 규칙을 적용한다.**

## 5. FAQ 데이터에 붙이면 좋은 메타데이터

기존 FAQ마다 아래 필드를 붙이면 온토로지와 연결이 쉬워집니다.

```json
{
  "faq_id": "PS-FAQ-045",
  "category": "눈성형",
  "question_type": "수술후관리",
  "risk_level": "medium",
  "related_procedure": ["눈성형", "쌍꺼풀", "눈매교정"],
  "related_symptoms": ["실밥", "붓기", "멍"],
  "action": "일반 안내",
  "escalation": "심한 통증, 출혈, 시야 이상이 있으면 즉시 병원 연결",
  "forbidden": ["정상이라고 단정", "회복기간 확정"],
  "answer": "눈성형 후 실밥 제거는 보통 수술 후 5~7일 전후에 진행되는 경우가 많습니다. 정확한 일정은 병원 안내를 따라 주세요."
}
```

## 6. 소규모 성형외과용 최소 버전

처음부터 복잡하게 만들 필요는 없습니다. 초기 버전은 아래 정도면 충분합니다.

```json
{
  "procedures": ["눈성형", "코성형", "필러", "보톡스", "리프팅", "지방흡입", "지방이식"],
  "question_types": ["예약", "비용", "수술전준비", "수술후관리", "서류", "환불", "부작용의심"],
  "high_risk_signals": ["심한 통증", "지속 출혈", "고열", "고름", "피부색 변화", "시야 이상", "호흡곤란"],
  "actions": {
    "low": "FAQ 자동 답변",
    "medium": "FAQ 답변 후 병원 문의 권장",
    "high": "즉시 병원 연락 또는 응급 안내"
  }
}
```

이 정도만 있어도 챗봇이 다음 차이를 구분할 수 있습니다.

| 질문 | 처리 |
|---|---|
| “오늘 상담 가능해요?” | 예약 안내 |
| “코수술 얼마예요?” | 예상 범위 + 상담 필요 |
| “쌍수 실밥 언제 빼요?” | 일반 수술 후 안내 |
| “눈수술 후 앞이 흐려요” | 즉시 병원 연락 |
| “필러 맞고 하얗게 됐어요” | 고위험, 즉시 연결 |

---

## 7. 최종 추천 구조

소규모 성형외과에는 아래 조합이 가장 현실적입니다.

```text
성형외과 FAQ MD 파일
+ 벡터 DB
+ FAQ별 메타데이터
+ 간단한 온토로지 JSON
+ 위험 질문 라우팅 규칙
```

이렇게 만들면 챗봇은 단순히 FAQ를 찾아주는 수준을 넘어갑니다.

> **예약 문의는 예약으로, 비용 문의는 상담 전환으로, 위험 증상은 즉시 병원 연결로 보내는 실전형 고객응대 시스템**이 됩니다.

---

## 8. 실제 적용 시 주의사항

1. 비용은 항상 **예상 범위**로만 안내합니다.
2. 수술 가능 여부는 챗봇이 판단하지 않습니다.
3. 사진만 보고 진단하지 않습니다.
4. 위험 증상은 “정상입니다”라고 답하지 않습니다.
5. 위험도 high 질문은 FAQ 답변보다 **즉시 병원 연락 안내**가 우선입니다.
6. 미성년자, 임신 가능성, 기저질환, 복용약 관련 질문은 상담실 또는 의료진 확인으로 연결합니다.
7. 환불, 예약금, 보험 적용 여부는 확정적으로 말하지 않습니다.
8. 모든 답변은 병원 내부 정책과 최신 안내문에 맞게 수정해서 사용합니다.
